# Monitoring

L'entreprise dans laquelle vous travaillez a mis en place des outils pour vérifier la qualité du site que vous concevez.

Appuyez-vous sur ces outils pour améliorer la qualité du site et apprendre de nouvelles façons de le faire.

## Vérifiez la validité du code HTML et CSS

Valider le code HTTP de votre site : https://validator.w3.org/

Valider le code CSS de votre site : https://jigsaw.w3.org/css-validator/

## Mesurer la qualité de votre site

Ces outils vous permettront d'identifier des optimisations possibles en terme
- d'accessibilité
- de performances
- de bonnes pratiques
- de référencement naturel (SEO)

Ne vous arrêtez pas aux scores, en lisant les rapports plus en détail, ceux-ci vous apprendront quoi faire pour corriger le problème la plupart du temps.

## Analyse périodique - Speedlify 

Une analyse des sites de chaque groupes de projet a lieu toutes les 30 minutes.

Elle est coupée la nuit pour éviter de consommer inutilement des ressources.

Vous trouverez le résultat de l'analyse de votre site et métriques et tendances de performance, accessibilité, bonnes pratiques et SEO à l'adresse suivante :

https://coda-school.github.io/speedlify-dijon-b1-2025/

## Si vous souhaitez que d'autres pages de votre site soient monitorées

Si votre site comporte plusieurs pages, **notifiez votre enseignant** pour qu'il puisse **l'ajouter à la liste des pages à analyser** pour votre site.

## Analyse ponctuelle

Si vous ne souhaitez pas attendre, vous pouvez utiliser Lighthouse :

- Dans les devtools des navigateurs basés sur Chrome
- En ligne sur votre site déployé : https://pagespeed.web.dev/
